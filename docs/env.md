# Environment & local config

This document shows where Gimble stores local configuration and what the common environment variables mean.

## Local config paths

- macOS: `~/Library/Application Support/gimble/`
- Linux: `~/.config/gimble/`

These directories contain your profiles, local cache, and any generated credentials used by the CLI.

## chat.env

When using a private Gimble Cloud backend, the CLI reads a small environment file (commonly named `chat.env`) containing connection values. Example:

```env
GIMBLE_CLOUD_API_BASE=https://chat.gimble.dev
GIMBLE_CLOUD_API_TOKEN=<auto-generated>
GIMBLE_DEVICE_ID=<auto-generated>
```

Notes:
- Keep `chat.env` out of version control. Do not commit credentials.
- `GIMBLE_CLOUD_API_TOKEN` is usually generated by the cloud/session setup and stored locally.

## Logs

- The CLI may write local session logs which are sanitized before upload when using cloud mode. If you need to inspect logs, look under the local config directory above (check for `session` or `logs` subfolders).

## Network / proxy

If your environment requires a proxy, export standard variables before running the CLI:

```bash
export HTTPS_PROXY="https://proxy.example:3128"
export HTTP_PROXY="http://proxy.example:3128"
```

Adjust these to your environment so the CLI's outbound HTTPS requests (for updates or cloud uploads) will succeed.
